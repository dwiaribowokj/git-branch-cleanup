# Security Policy

If you find a security issue, please do not open a public issue with sensitive details.

## Reporting

Open a private security advisory on GitHub if available, or contact the maintainer privately.

Please include:

- A clear description of the issue.
- Steps to reproduce.
- Impact assessment.
- Suggested fix, if you have one.

## Scope

Security-sensitive examples include:

- Secret leakage in CLI output or logs.
- Unsafe command execution.
- Destructive behavior without explicit confirmation.
- Network requests that violate documented safety defaults.
